## 业务背景

**商家智能体**：C 端用户访问商家页面后开启对话，服务侧由智能体代替真人客服。

核心目标：**获取用户线索（留资/套电）**

对话策略：
- 理解用户意图（咨询产品/服务、比价、了解详情等）
- 结合 RAG 知识库（商家资料）准确回答问题
- 识别高意向用户，适时引导留资/预约

## 现阶段目标

1. 业务层改写：将商旅业务逻辑替换为商家智能体业务逻辑
   - RAG 知识库内容：差旅标准 → 商家服务/产品资料
   - 记忆系统：出行偏好 → 用户意向、对话进度
   - 意图识别：出行意图 → 咨询意图 + 套电时机判断
   - 新增套电策略 skill
2. 接入层：FastAPI HTTP 层，对接 Next.js 前端

## 项目关键约定

- Agent 间通信格式：`Msg` 对象，`content` 字段是 JSON 字符串
- 存储现状：短期记忆 = Python 内存 list，长期记忆 = JSON 文件
- LLM：DeepSeek（config.py 里配置，API Key 待改成环境变量）
